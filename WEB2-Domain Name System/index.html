host와 host가 서로 통신을 하기 위해서는 서로에 대한 주소가 필요합니다. 인류에서 사용하는 주소를 ip address라고 합니다.

ip address 덕분에 인류는 host와 host를 연결해서 통신을 할 수 있게 되었고 이것은 한대의 컴퓨터가 한대의 한계를 추월해버리는 위대한 순간이다.

WEB2-Domain Name system-14.수업을 마치며

  • HOST에게 자신의 이름 Domain Name을 지정해 주는것을 배웠습니다.
  • IP주소를 변경해도 서비스를 계속 제공받을수 있는 유연함을 얻게 되었다.
  • 앞으로 Domain Name과 관련해서 공부해야 될 키워드를 알려준다 한다.
  • DNS 최초의 서버인 BIND. 여러가지 DNS 서버가 많다.
  • 내 컴퓨터에 연결된 Local DNS Server부터 server에 DNS IP를 알려주는 DNS서버까지 다양한 것들이 존재합니다.
  • 여러 서비스를 비교해가면서 어떤 서비스가 나에게 적합한지를 골라보는것도 좋을거같다.
  • 집에서 서버를 운영하고 싶다면 - Dynamic DNS라는 것에 도전해보자 집으로 들어오는 인터넷에 IP는 변경됩니다. IP는 귀한 자원입니다. 노트북이라면 더욱 자주 변경됩니다.
  • Dynamic DNS(DDNS) 서버가 설치된 IP 주소가 변경되었을 때 Domain Name을 관리하는 server에게 변경된 IP가 전달됩니다. 사람들은 Domain Name을 이용해서 나의 컴퓨터에 접속하기 때문에 서버의 끊김을 최소화 하면서도 서버를 자신의 가정에서 IP가 계속 변경되는 환경에서도 운영할수 있습니다.
  • web server를 운영하는 경우라면 통신을 할때 http가 아닌 https를 사용하므로서 사이트를 누군가가 속여서 피싱을 한 나쁜행위를 차단할 수 있습니다. 이런 기술을 HTTPS 또는 SSL이라고 합니다.
  • 여러분이 SSL을 이용하면 SSL 인증서를 발급 받게 되는데 이 인증서를 사용하면 사용자가 위조된 웹사이트를 접속했을 때 웹 브라우저가 접속을 차단하고 강력한 경고 메세지를 사용자에게 보여줍니다.
  • 최근에는 인증서가 무료화 되고 있는 추세이고 매우 간편하게 설치 할 수 있는 방향으로 진화하고 있기 때문에 사용하지 않을 이유가 없습니다.
